1. Projektübersicht
Secury ist ein moderner, browserbasierter Passwort Manager mit integriertem Passwort Generator. Das Projekt wird als OpenSource Web-Anwendung entwickelt und dient gleichzeitig als persönliches Produktivitätstool und Portfolio-Projekt.

Der Kerngedanke von Secury ist Sicherheit durch Zero-Knowledge Verschlüsselung: Passwörter werden ausschliesslich client-seitig mit AES-256 verschlüsselt, bevor sie in der Datenbank gespeichert werden. Der Server sieht zu keinem Zeitpunkt unverschlüsselte Passwörter.

1.1 Projektziele
    • Eigener, sicherer Passwort Manager für den täglichen Gebrauch
    • Portfolio-Projekt das Sicherheitskompetenz demonstriert
    • OpenSource Veröffentlichung auf GitHub
    • Grundlage für eine spätere iOS-App
    • Erfahrung mit Zero-Knowledge Kryptographie und Supabase

1.2 Zielgruppe
    • Primär: Leon selbst (persönliches Tool)
    • Sekundär: IT-affine Nutzer die eine einfache OpenSource-Lösung suchen
    • Langfristig: Allgemeine Nutzer nach iOS-Release

3. Sicherheitsarchitektur (Zero-Knowledge)
Das Herzstück von Secury ist das Zero-Knowledge Prinzip. Kein Server, keine Datenbank und kein Dritter hat jemals Zugriff auf unverschlüsselte Passwörter. Die gesamte Kryptographie findet ausschliesslich im Browser des Nutzers statt.

3.1 Verschlüsselungsablauf
Beim Speichern eines Passworts:
    • Nutzer gibt Passwort-Eintrag ein
    • Browser generiert aus dem Master-Passwort einen AES-256 Schlüssel (PBKDF2)
    • Eintrag wird client-seitig mit AES-256-GCM verschlüsselt
    • Nur der verschlüsselte Blob wird an Supabase gesendet
    • Supabase speichert ausschliesslich verschlüsselte Daten

Beim Abrufen eines Passworts:
    • Nutzer ist eingeloggt und gibt Master-Passwort ein
    • Browser lädt verschlüsselten Blob von Supabase
    • AES-256 Schlüssel wird erneut aus Master-Passwort abgeleitet
    • Entschlüsselung findet lokal im Browser statt
    • Klartextpasswort wird nur im Arbeitsspeicher gehalten, nie gespeichert

3.2 Wichtige Sicherheitshinweise
    • Das Master-Passwort wird NIEMALS gespeichert oder übertragen
    • Geht das Master-Passwort verloren, sind alle Einträge unwiederbringlich verloren
    • Empfehlung: Master-Passwort sicher offline aufbewahren
    • Supabase Row Level Security (RLS) verhindert Zugriff anderer Nutzer


4. Features — Version 1.0
4.1 Authentifizierung
    • Registrierung mit E-Mail und Passwort (Supabase Auth)
    • Login / Logout
    • Session-Management (automatisch eingeloggt bleiben)
    • Benutzerseite / Profil (Account-Infos, Master-Passwort ändern)

4.2 Passwort-Verwaltung
    • Passwort-Eintrag erstellen (Name, Benutzername, Passwort, URL, Kategorie, Notizen)
    • Einträge anzeigen in einer übersichtlichen Liste
    • Eintrag bearbeiten
    • Eintrag löschen (mit Bestätigung)
    • Passwort anzeigen / verstecken Toggle
    • Passwort mit einem Klick in die Zwischenablage kopieren

4.3 Passwort Generator
    • Passwortlänge einstellbar (8 bis 64 Zeichen)
    • Optionen: Grossbuchstaben, Kleinbuchstaben, Zahlen, Sonderzeichen
    • Passwort direkt in neuen Eintrag übernehmen
    • Stärke-Anzeige (Weak / Fair / Strong / Very Strong)

4.4 Organisation & Suche
    • Kategorien: Social Media, Banking, E-Mail, Gaming, Shopping, Arbeit, Sonstiges
    • Suchfunktion (Echtzeit-Filterung nach Name / URL)
    • Filterung nach Kategorie

4.5 Geplant für spätere Versionen
    • Browser Extension für automatisches Ausfüllen
    • Import / Export (CSV, JSON)
    • Zwei-Faktor-Authentifizierung (2FA)
    • iOS-App (React Native oder Swift)
    • Passwort-Sharing (verschlüsselt)
    • Sicherheits-Audit (doppelte Passwörter, schwache Passwörter erkennen)
